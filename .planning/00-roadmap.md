# CLI-Recover v2.0 Roadmap

## 핵심 철학
- **Isolation First**: 완전히 격리된 구현
- **Simplicity**: 단순함이 복잡함을 이긴다
- **TDD**: 테스트 먼저, 구현은 나중에
- **No Premature Abstraction**: 필요할 때까지 추상화하지 않음

## 미래 계획 (v2.1+)

### 고려 사항
- TUI 래퍼 (선택적)
- 추가 Provider (요청 시)
- 플러그인 시스템 (필요 시)
- 클라우드 스토리지 (수요 기반)

### 원칙
- 각 기능은 완전히 격리
- 기존 코드 영향 없음
- 사용자 요청 기반
- 복잡도 관리 우선

## 타임라인
- **총 기간**: 약 2주
- **일일 목표**: 작고 완성된 기능
- **주간 목표**: 사용 가능한 도구
- **최종 목표**: 단순하고 안정적인 백업 도구

## 위험 관리
- **복잡도 상승**: 매일 복잡도 체크
- **기능 과다**: 핵심 기능만 구현
- **추상화 유혹**: v2.0에서는 금지
- **일정 지연**: 기능 축소로 대응

## 성공의 정의
- 사용자가 매뉴얼 없이 사용 가능
- 개발자가 코드 즉시 이해
- 버그 리포트 < 5개/월
- 복잡도 < 20/100 유지

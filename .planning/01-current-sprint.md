## 로그 파일 시스템 구조
```
~/.cli-recover/logs/
├── metadata/         # 메타데이터 JSON
│   └── *.json       # 각 작업별 메타데이터
└── files/           # 실제 로그 파일
    └── backup/      # 백업 로그
    └── restore/     # 복구 로그
```

## 주요 기능
- 각 백업/복구 작업마다 고유 ID 생성
- 상세 로그 파일 자동 생성
- 작업 상태 추적 (running, completed, failed)
- 메타데이터 저장 (namespace, pod, path 등)
- 오래된 로그 자동 정리 (logs clean)
